# Workflow Router Test Cases

Use these cases to validate routing behavior manually or from a local router test harness.

## Cases

### Python bug fix

Prompt: "Fix failing pytest around user creation."

Expected skills:

- debugging
- testing-tdd
- python

### Broad architecture review

Prompt: "Review this module architecture and suggest how to split it."

Expected skills:

- engineering-standards
- architecture-principles
- code-quality
- code-review
- relevant ecosystem skill if detectable

### Large file refactor

Prompt: "This file is 3000 lines, refactor it safely."

Expected skills:

- engineering-standards
- refactoring
- code-quality
- testing-tdd
- relevant ecosystem skill

### Security-sensitive API change

Prompt: "Add authorization checks to this endpoint."

Expected skills:

- security
- testing-tdd
- architecture-principles if boundaries matter
- relevant ecosystem skill

### ESPHome relay automation

Prompt: "Create ESPHome config for relay and sensor fail-safe behavior."

Expected skills:

- homeassistant if HA entity behavior is involved
- esphome
- esp32
- testing-tdd or validation strategy

### TODO plan

Prompt: "Write a TODO plan for splitting this package."

Expected skills:

- engineering-standards
- architecture-principles
- refactoring
- documentation
