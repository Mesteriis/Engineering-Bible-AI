# Code of Conduct

## Expected Behavior

Participants are expected to:

- be direct and respectful;
- focus criticism on technical content and project outcomes;
- assume good intent while requiring evidence for technical claims;
- keep discussions safe for public OSS collaboration;
- avoid sharing private, sensitive, or credential-bearing information.

## Unacceptable Behavior

Unacceptable behavior includes:

- harassment, threats, or personal attacks;
- publishing private information without permission;
- intentionally introducing secrets or unsafe runtime instructions;
- repeated off-topic disruption;
- bad-faith security disclosure.

## Enforcement

Maintainers may edit, hide, lock, or remove issues, pull requests, comments, or
contributors that violate this code of conduct.

Security-sensitive reports should follow `SECURITY.md`.
