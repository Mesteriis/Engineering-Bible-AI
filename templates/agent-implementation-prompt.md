# Prompt: аккуратно внедрить engineering rules в мой Codex

Ты работаешь в моём локальном Codex окружении. Нужно аккуратно внедрить пакет инженерных правил из архива `codex-engineering-rules-v2.zip`.

## Цель

Установить/обновить мои глобальные Codex-инструкции, skills, reference-доки, шаблоны и скрипты так, чтобы:

- существующие локальные настройки не были потеряны;
- мой `workflow-router` остался обязательной точкой входа для нетривиальных инженерных задач;
- root `AGENTS.md` остался технологически нейтральным;
- языковые правила жили в отдельных skills;
- общие инженерные принципы жили в `engineering/`;
- внедрение было проверено командами.

## Жёсткие требования

- Не удаляй мои существующие файлы без явного подтверждения.
- Перед изменениями покажи, какие файлы будут добавлены, изменены или потенциально конфликтуют.
- Если файл уже существует, сначала сравни содержимое.
- Для конфликтующих файлов предложи безопасную стратегию: merge, backup или skip.
- Не перетирай мой локальный `workflow-router`, если он отличается; сначала покажи diff и предложи merge.
- Не добавляй языковые правила в root `AGENTS.md`; они должны остаться в skills.
- Не заявляй, что проверка прошла, если команда не запускалась.

## Ожидаемая структура

После внедрения должны быть доступны:

```text
~/.codex/AGENTS.md
~/.codex/engineering/
~/.codex/skills/
~/.codex/templates/
~/.codex/scripts/
```

Если моя установка Codex использует другой путь, сначала определи его из текущего окружения и явно сообщи.

## Порядок работы

1. Найди текущую Codex-директорию.
2. Распакуй архив во временную директорию.
3. Покажи краткий inventory архива.
4. Проверь текущие файлы назначения.
5. Найди конфликты.
6. Для каждого конфликта предложи стратегию:
   - `merge` для совместимых markdown-инструкций;
   - `backup+replace` для старых версий без локальных изменений;
   - `skip` если файл явно кастомный и нельзя безопасно объединить.
7. Создай backup перед перезаписью любых файлов.
8. Скопируй новые файлы.
9. Сохрани права исполнения для скриптов.
10. Запусти валидацию.
11. Отчитайся.

## Команды для проверки

Запусти доступные команды:

```bash
bash ~/.codex/scripts/validate-installed-tree.sh ~/.codex ~/.agents
python ~/.codex/scripts/check-file-size.py ~/.codex --hard 10000
```

Если путь отличается, адаптируй команды.

## Финальный отчёт

Ответ должен быть в таком формате:

```markdown
Changed files:
- ...

Backups:
- ...

Conflicts:
- ...

Summary:
- ...

Validation:
- Ran: ...
- Result: ...

Risks:
- ...
```

Если были конфликты, не скрывай их.

Если какие-то проверки не были выполнены, укажи точную причину.
